# 🍔 TaiBites — Food Delivery App (Front-End)

> Hungry? Browse restaurants → open a menu → add to cart → login with Google → order placed. 🛵

🔗 **Live Demo:** [https://GH-IRIS.github.io/TAIBITES/](https://GH-IRIS.github.io/TAIBITES/) · **Code:** [https://github.com/GH-IRIS/TAIBITES](https://github.com/GH-IRIS/TAIBITES)

## 🗺 The user journey
🏠 **Landing page** → 🍽 **Restaurant listing** (Veg/Non-veg filter) → 📋 **Menu page** → 🛒 **Cart** (items + delivery fee + grand total) → 🔐 **Google login (Clerk)** → ✅ **Order confirmed!**

## ✨ Highlights
- Add to cart **works from every restaurant's menu**
- Cart page shows quantities, delivery fee & grand total
- **Login with Google (Clerk) is compulsory before placing an order**
- Restaurant filters: cuisine, rating, Veg / Non-veg
- 100% responsive — designed mobile-first like a real food app

## 🛠 Tech Stack
HTML5 · CSS3 · Clerk (Google auth) · AI-assisted features (Antigravity Code Assistant) · GitHub Pages

## 🤖 How I used AI
I designed the HTML structure and Swiggy-style visual branding; AI helped create the restaurant database arrays, coded the cart single-restaurant restriction logic, and engineered the interval timers for the live delivery tracking timeline.

## 📚 What I learned
I learned to manage complex state structures (validating that users only buy from one restaurant at a time), set up event-driven UI updates, and implemented active CSS timeline animations.
